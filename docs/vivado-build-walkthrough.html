kaipu-ooo · mac-native build flow

Vivado Build Walkthrough

Synthesise RTL for the Digilent Nexys A7-100T (Artix-7 XC7A100T) using Vivado 2026.1 inside a Docker container. Flash directly from macOS with openFPGALoader — no Vivado license or Linux VM on the host.

Prerequisites — one-time setup, already complete
Docker Desktop running — disk on /Volumes/FD_4TB, memory ≥ 16 GB
kaipu-vivado image built via make docker-build
oss-cad-suite at ~/Documents/FPGA_TOOLS/oss-cad-suite/bin/
Every-build workflow
01
Build the bitstream

Run from the kaipu-ooo directory. Vivado runs inside the container; the repo root is mounted at /work so it can read RTL sources and write the bitstream back to the host. The container tears itself down when done.

terminal
cd ~/Documents/GitHub/kaipu-ooo
make docker-bit

Output lands at synth/nexys_a7/top.bit on the host. Typical runtime: 15–30 min depending on design complexity.

To build with a specific firmware instead of the default looping.asm no-op:

with custom firmware
make docker-bit PROG=vendor/kaipu/tests/programs/foo.asm
What's in the container: Vivado 2026.1, Python 3, Make. The repo is bind-mounted read-write at /work — build artifacts written there appear on your host immediately.
bitstream ready at synth/nexys_a7/top.bit
02
Connect the board

Plug the Nexys A7 into a USB port. The board uses an FT2232H onboard JTAG chip (Digilent HS2-compatible). openFPGALoader from oss-cad-suite handles it natively — no Digilent drivers needed.

To verify the board is detected before flashing:

optional detection check
~/Documents/FPGA_TOOLS/oss-cad-suite/bin/openFPGALoader --detect
UART demos: the board exposes two /dev/tty.usbserial-* endpoints — one for JTAG, one for UART. When opening a serial terminal, pick the correct one. The glob may land on the JTAG endpoint, which gives no echo.
board detected
03
Flash — run in your own terminal

The Claude Code agent sandbox cannot reach USB/JTAG devices. This step must be run by you in a separate terminal.

your terminal (not agent sandbox)
# short form — uses FPGA_TOOLS path from Makefile
make flash

# or explicitly
~/Documents/FPGA_TOOLS/oss-cad-suite/bin/openFPGALoader \
  -c digilent_hs2 synth/nexys_a7/top.bit

In Claude Code you can prefix with ! to run it inline: type ! make flash in the prompt and the output lands directly in the conversation.

Flash takes a few seconds. The board resets automatically and begins executing the loaded firmware.
Maintenance
Auth token expired

The AMD auth token (docker/.amd_token) is valid for ~1 week. If make docker-build fails with an auth error, regenerate it:

interactive — prompts AMD credentials
make docker-auth-token
Rebuild Docker image

After a Vivado version bump or if the image is lost:

~10 GB download, ~90 min
make docker-auth-token  # if token stale
make docker-build
Docker Desktop won't start

bbolt database panic after a reset: click Reset to factory defaults in the Docker Desktop error dialog. Then in Settings → Resources → Advanced:

Set disk image location → /Volumes/FD_4TB/Docker/DockerDesktop/, memory → 16 GB+. Apply & Restart.

/var/run/docker.sock missing

Happens after a Docker Desktop reset — the symlink disappears. The Makefile auto-detects and falls back to ~/.docker/run/docker.sock automatically. No action needed.

To fix permanently: Docker Desktop → Settings → Advanced → Allow the default Docker socket to be used.
